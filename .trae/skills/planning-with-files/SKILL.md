# Planning with Files Skill

## 技能名称
planning-with-files

## 技能描述
使用持久化 Markdown 文件进行规划、进度跟踪和知识存储的工作流程模式，这是 Meta 以 20 亿美元收购 Manus 的核心工作模式。

## 功能

### 1. 规划命令
- **命令**：/plan
- **描述**：创建或更新规划文件，用于任务分解和进度跟踪
- **示例**：
  ```
  /plan
  # 任务：实现用户认证系统
  - [ ] 设计数据库模型
  - [ ] 实现登录接口
  - [ ] 实现注册接口
  - [ ] 添加 JWT 验证
  - [ ] 测试认证流程
  ```

### 2. 开始命令
- **命令**：/start
- **描述**：启动规划会话，初始化工作流程
- **示例**：
  ```
  /start
  # 开始新的规划会话
  项目：AI 辅助设计工具
  目标：实现自动命名功能
  ```

## 工作流程

1. **初始化**：使用 `/start` 命令启动规划会话
2. **规划**：使用 `/plan` 命令创建详细的任务计划
3. **执行**：按照计划执行任务，更新进度
4. **跟踪**：使用 Markdown 文件跟踪进度和记录知识
5. **完成**：标记任务完成，总结成果

## 模板

### 任务计划模板
```markdown
# 任务计划：{{任务名称}}

## 目标
{{任务目标}}

## 任务分解
- [ ] {{任务1}}
- [ ] {{任务2}}
- [ ] {{任务3}}

## 时间线
- 开始时间：{{开始时间}}
- 预计完成时间：{{预计完成时间}}

## 资源
{{所需资源}}

## 风险
{{潜在风险}}

## 进度更新
- [YYYY-MM-DD] {{更新内容}}
```

### 进度跟踪模板
```markdown
# 进度跟踪：{{项目名称}}

## 当前状态
- 总任务数：{{总任务数}}
- 已完成：{{已完成任务数}}
- 进度：{{进度百分比}}%

## 最近更新
- [YYYY-MM-DD HH:MM:SS] {{更新内容}}

## 待办事项
- [ ] {{任务1}}
- [ ] {{任务2}}

## 已完成事项
- [x] {{任务1}}
- [x] {{任务2}}

## 遇到的问题
{{问题描述}}

## 解决方案
{{解决方案}}
```

### 发现记录模板
```markdown
# 发现记录：{{项目名称}}

## 日期
[YYYY-MM-DD HH:MM:SS]

## 发现内容
{{发现描述}}

## 影响
{{影响分析}}

## 建议
{{建议方案}}

## 相关资源
{{相关链接或参考资料}}
```

## 配置

### 目录结构
```
.trae/
  skills/
    planning-with-files/
      SKILL.md
      templates/
        task_plan.md
        progress.md
        findings.md
      scripts/
        init-session.ps1
        check-complete.ps1
```

### 模板文件
- `templates/task_plan.md`：任务计划模板
- `templates/progress.md`：进度跟踪模板
- `templates/findings.md`：发现记录模板

### 脚本文件
- `scripts/init-session.ps1`：初始化规划会话
- `scripts/check-complete.ps1`：检查任务完成情况

## 使用说明

1. **安装**：将技能目录放置在 `.trae/skills/` 目录下
2. **激活**：在 TRAE 中使用 `/plan` 或 `/start` 命令
3. **配置**：根据需要修改模板文件
4. **使用**：按照工作流程执行任务并跟踪进度

## 优势

- **持久化**：使用本地文件存储，不会丢失上下文
- **结构化**：使用 Markdown 格式，结构清晰易读
- **灵活**：可根据项目需求自定义模板和工作流程
- **可追溯**：完整记录项目进展和决策过程
- **知识管理**：自动积累项目知识和经验

## 适用场景

- **复杂项目**：需要详细规划和跟踪的大型项目
- **团队协作**：多人参与的协作项目
- **长期项目**：持续时间较长的项目
- **知识密集型项目**：需要大量知识管理的项目
